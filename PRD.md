## 2. 产品目标与背景

### 背景

微信公众号编辑器不支持 Markdown 语法，内容创作者需要手动调整排版样式，费时费力。市面虽有类似工具（如 Markdown Nice），但多数依赖外部服务或功能臃肿。

### 目标

提供一个**纯前端、零依赖**的 Markdown 转公众号排版工具，支持一键复制富文本粘贴到公众号后台，大幅提升内容创作效率。

### 成功标准

- 用户能在 30 秒内完成 Markdown → 公众号排版 → 复制粘贴的完整流程
- 粘贴到公众号编辑器后格式保持一致，无错位
- 支持 5+ 套预设主题（简约/科技/文艺/二次元/商务），可微调样式参数

## 3. 用户角色与使用场景

### 角色

| 角色 | 描述 |
|------|------|

### User Story

**US-01:** 作为内容创作者，我希望在左侧输入 Markdown，右侧实时看到公众号排版效果，以便快速调整内容。

**US-02:** 作为内容创作者，我希望切换不同的排版主题，以便适配不同风格的公众号文章。

**US-03:** 作为内容创作者，我希望微调字号、行间距、主题色等参数，以便排版更符合我的品牌风格。

**US-04:** 作为内容创作者，我希望一键复制排版后的富文本，直接粘贴到公众号编辑器即可发布。

**US-05:** 作为内容创作者，我希望关闭页面后再打开，之前编辑的内容和样式设置还在。

## 4. 功能需求

### F01: 左右分栏编辑器

**描述：** 页面分为左右两栏，左栏为 Markdown 文本编辑器，右栏为公众号排版预览区域。

**AC：**

- **AC-F01-01:** Given 页面加载完成, When 用户查看页面, Then 左右分栏布局显示，左侧为 Markdown 编辑区，右侧为预览区
- **AC-F01-02:** Given 左侧编辑器有内容, When 用户修改 Markdown 文本, Then 右侧预览区实时更新渲染结果（延迟 ≤ 300ms）
- **AC-F01-03:** Given 首次打开（无 localStorage）, When 页面加载, Then 编辑器显示内置示例 Markdown 文章

### F02: Markdown 语法支持

**描述：** 支持常用 Markdown 语法，正确渲染为公众号兼容的 HTML。

**支持的语法：**

- 标题（h1-h6）
- 加粗、斜体、删除线
- 有序列表、无序列表
- 引用（blockquote）
- 代码块（带语法高亮）和行内代码
- 图片（img 标签）
- 链接（公众号不支持外链，需转为脚注样式）
- 表格

**AC：**

- **AC-F02-01:** Given Markdown 中包含 h1-h6 标题, When 渲染预览, Then 各级标题样式清晰区分，符合当前主题风格
- **AC-F02-02:** Given Markdown 中包含加粗/斜体/删除线, When 渲染预览, Then 正确显示对应样式
- **AC-F02-03:** Given Markdown 中包含有序/无序列表, When 渲染预览, Then 列表缩进正确，序号/符号显示正常
- **AC-F02-04:** Given Markdown 中包含引用块, When 渲染预览, Then 引用块有左边框和背景色，符合当前主题
- **AC-F02-05:** Given Markdown 中包含代码块（指定语言）, When 渲染预览, Then 代码块有背景色、等宽字体，带语法高亮
- **AC-F02-06:** Given Markdown 中包含行内代码, When 渲染预览, Then 行内代码有背景色和等宽字体
- **AC-F02-07:** Given Markdown 中包含图片语法 `![alt](url)`, When 渲染预览, Then 图片正确显示，居中，自适应宽度
- **AC-F02-08:** Given Markdown 中包含链接 `[text](url)`, When 渲染预览, Then 链接文本带特殊颜色标记，URL 以脚注形式显示在文末
- **AC-F02-09:** Given Markdown 中包含表格, When 渲染预览, Then 表格样式美观，有边框/斑马纹，符合当前主题

### F03: 主题切换

**描述：** 预设多套公众号排版风格，用户可一键切换。

**预设主题（至少 5 套）：**

| 主题 | 风格描述 |
|------|---------|
| 简约 | 黑白灰为主，干净清爽，适合通用文章 |
| 科技 | 深色调/蓝色系，适合技术类文章 |
| 文艺 | 暖色调，适合生活/文化类文章 |
| 二次元 | 活泼配色（粉/紫/蓝渐变），圆角卡片，可爱风装饰元素，适合 ACG/动漫类文章 |
| 商务 | 深蓝/藏青 + 金色点缀，衬线标题，严谨排版，适合行业报告/商业分析类文章 |

**AC：**

- **AC-F03-01:** Given 页面加载, When 用户查看主题选择区域, Then 显示至少 5 套预设主题供选择
- **AC-F03-02:** Given 当前主题为 A, When 用户选择主题 B, Then 预览区立即切换为主题 B 的样式
- **AC-F03-03:** Given 用户选择了主题, When 刷新页面, Then 主题选择保持不变（localStorage 持久化）

### F04: 样式微调

**描述：** 支持细粒度的样式参数调整。

**可调参数：**

| 参数 | 范围 | 默认值 |
|------|------|--------|
| 正文字号 | 14-18px | 16px |
| 行间距 | 1.5-2.5 | 1.75 |
| 主题色 | 颜色选择器 | 跟随主题 |
| 代码块配色 | 预设方案 | 跟随主题 |

**AC：**

- **AC-F04-01:** Given 样式微调面板可见, When 用户调整正文字号, Then 预览区字号实时变化
- **AC-F04-02:** Given 样式微调面板可见, When 用户调整行间距, Then 预览区行间距实时变化
- **AC-F04-03:** Given 样式微调面板可见, When 用户通过颜色选择器修改主题色, Then 预览区标题、链接等元素颜色实时变化
- **AC-F04-04:** Given 样式微调面板可见, When 用户切换代码块配色方案, Then 预览区代码块配色实时变化
- **AC-F04-05:** Given 用户调整了样式参数, When 刷新页面, Then 样式参数保持不变（localStorage 持久化）

### F05: 一键复制

**描述：** 一键复制预览区的富文本，粘贴到公众号编辑器可直接使用。

**AC：**

- **AC-F05-01:** Given 预览区有渲染内容, When 用户点击"复制"按钮, Then 富文本复制到剪贴板，提示"复制成功"
- **AC-F05-02:** Given 已复制富文本, When 用户粘贴到微信公众号编辑器, Then 排版样式与预览一致（标题、列表、代码块、表格等均正确）
- **AC-F05-03:** Given 复制成功, When 查看复制内容, Then 样式为内联 CSS（非 class），确保公众号兼容

### F06: 内置示例文章

**描述：** 预置一篇示例 Markdown 文章，涵盖所有支持的语法元素。

**AC：**

- **AC-F06-01:** Given 首次打开页面（无 localStorage）, When 页面加载, Then 编辑器显示内置示例文章
- **AC-F06-02:** Given 示例文章已加载, When 查看预览区, Then 能看到标题、列表、代码块、表格、图片、引用等所有元素的渲染效果

## 5. 业务规则与边界条件

| 编号 | 规则 |
|------|------|
| BR-01 | 公众号不支持外链，所有 `[text](url)` 链接需转为脚注样式（正文显示 text + 上标序号，文末列出 URL） |
| BR-02 | 代码块需使用内联样式，不依赖外部 CSS |
| BR-03 | 图片使用 `<img>` 标签，最大宽度 100%，居中显示 |
| BR-04 | 所有样式必须为 inline style，不使用 class（公众号会过滤 class） |
| BR-05 | localStorage 存储上限注意不超过 5MB |

## 6. 异常与错误设计

| 场景 | 处理方式 |
|------|---------|
| Markdown 语法错误 | 容错渲染，不崩溃，尽可能显示 |
| 剪贴板 API 不可用 | 降级为选中文本 + 提示手动复制 |
| localStorage 已满 | 提示用户清理浏览器存储 |
| 图片 URL 无法加载 | 显示 alt 文本 + 占位框 |

## 8. 视觉风格与设计规范（v1.3 重写）

> **设计哲学：极简但有设计感。** 不是"什么都不加"的朴素，而是"每个细节都推敲过"的克制。像 Notion、Linear、Vercel Dashboard 的风格——打开一看就知道这是设计过的，但说不出具体"设计"在哪里。

### 8.1 设计系统基础

#### 8px 网格

所有间距基于 8px 倍数：4/8/12/16/24/32/48。不允许出现 5px/7px/15px/23px 等非网格值。

#### 字体层级（Type Scale）

| 层级 | 用途 | 字号 | 字重 | 颜色 |
|------|------|------|------|------|
| Display | 产品名 | 16px | 600 | `#1a1a2e` |
| Body | 正文/编辑器 | 14px | 400 | `#24292f` |
| Caption | 状态栏/辅助 | 12px | 400 | `#8b949e` |
| Label | 控件标签 | 13px | 500 | `#57606a` |

#### 颜色系统

| 角色 | 颜色 | 用途 |
|------|------|------|
| Primary | `#2563eb` | 唯一点缀色——按钮、选中态、焦点环、链接 |
| Primary Hover | `#1d4ed8` | 按钮 hover |
| Surface | `#ffffff` | 卡片、面板背景 |
| Background | `#f8f9fa` | 页面大底色 |
| Editor BG | `#fafbfc` | 编辑器背景（比白稍暗，区分预览区） |
| Border | `#e5e7eb` | 所有边框/分割线 |
| Text Primary | `#24292f` | 正文 |
| Text Secondary | `#57606a` | 标签、次要文字 |
| Text Muted | `#8b949e` | 状态栏、辅助信息 |
| Success | `#16a34a` | 复制成功 |
| Danger | `#dc2626` | 错误 |

**规则：整个页面只有一个彩色（Primary `#2563eb`），其余全是黑白灰。**

#### 圆角

统一 `8px`。所有可点击元素（按钮、下拉、卡片）同一圆角值，不混用。

#### 阴影

| 层级 | 值 | 用途 |
|------|-----|------|
| sm | `0 1px 2px rgba(0,0,0,0.05)` | 工具栏、状态栏 |
| md | `0 2px 8px rgba(0,0,0,0.08)` | 预览卡片、下拉菜单 |
| focus | `0 0 0 3px rgba(37,99,235,0.15)` | 焦点环（点缀色15%透明度） |

#### 过渡

所有交互状态统一 `transition: all 150ms ease`。不快不慢，不用花哨的缓动函数。

| 属性 | 值 |

| 底部 | `1px solid #e5e7eb` |

| 属性 | 值 |

| 外观 | 白色圆角矩形，`border: 1px solid #e5e7eb`，`border-radius: 8px`，`padding: 6px 12px` |
| 文字 | 当前主题名，13px/500/#24292f |
| 右侧 | 小三角 chevron 图标（用 CSS border 画，`#8b949e`） |
| Hover | `border-color: #d1d5db`，`background: #f9fafb` |
| 展开 | 下拉面板 `background: #fff`，`border-radius: 8px`，`shadow: md`，`border: 1px solid #e5e7eb` |
| 选项 | 每行显示主题名 + 一句话描述（如"简约 — 黑白灰，干净清爽"），`padding: 8px 12px` |
| 选项 Hover | `background: #f3f4f6` |
| 选中项 | 左侧显示 `✓`，文字颜色 `#2563eb` |
| 选项前缀 | 每个主题名前加一个 8px 小色点（圆形），颜色对应主题色 |

### 8.11 视觉验收标准

1. **设计感测试：** 打开页面，感觉是"有人设计过的"，而不是"默认 HTML 样式"。
2. **克制测试：** 页面上没有任何渐变、毛玻璃、动画（hover 颜色变化除外）。
3. **一致性测试：** 所有圆角统一 8px，所有间距是 8 的倍数，所有边框颜色相同。
4. **唯一彩色测试：** 除了主题本身的配色外，工具 UI 只有一个彩色（`#2563eb`），其余全是黑白灰。
5. **字体层级测试：** 页面上最多 4 种字号（16/14/13/12），不多不少。
6. **主题选择器测试：** 是自定义样式的下拉，不是原生 `<select>`，每个选项有主题名+描述+色点。
7. **预览卡片测试：** 白色卡片 + 8px 圆角 + 柔和阴影，没有微信外壳装饰。
